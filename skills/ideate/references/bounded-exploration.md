# Bounded Exploration ("Divergence Control")

**Core operation:** Manage tension between staying on-track and productive wandering by explicitly calibrating divergence levels.

## The Sweet Spot

**30-50% divergence** for effective exploration.

- **Below 30%:** Too constrained, missing opportunities
- **Above 50%:** Too scattered, losing coherence
- **At 0%:** Rigid, no discovery
- **At 100%:** Chaos, no progress

## Key Mechanisms

1. **Anchor** to a core problem/goal (the non-negotiable)
2. **Measure** how far current work diverges from anchor
3. **Establish** coherence boundaries ("wild but not that wild")
4. **Conduct** regular alignment checks
5. **Deliberately introduce** unexplored directions when too converged

## Applications

| Context | How to Apply |
|---------|--------------|
| **Research** | Define what's "in scope" explicitly so you know when you're wandering productively vs. procrastinating |
| **Meetings** | "Let's let this diverge for 10 minutes, then re-anchor" |
| **Innovation** | Create designated "wild zones" where anything goes, separate from core work |
| **Collaboration** | Agree on immovable anchors while allowing maximum divergence elsewhere |

## Anchor Types

- **Goal anchor:** "We must ship X by date Y"
- **Constraint anchor:** "Budget is fixed at $Z"
- **Quality anchor:** "Must meet these standards"
- **Scope anchor:** "Only solving for user segment A"

## Wild Zone Protocol

1. Explicitly designate time/space for divergence
2. Remove judgment during wild zone
3. Hard stop at boundary
4. Harvest insights, discard noise
5. Re-anchor before continuing

## Red Flags

- **Too converged:** Everything feels predictable, no surprises
- **Too diverged:** Can't explain how current work connects to goal
- **No anchors:** Drifting without knowing it
- **All anchors:** Can't move, everything feels constrained
