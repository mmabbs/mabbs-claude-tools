# State-Task Matching ("Cognitive Variability")

**Core operation:** Recognize four cognitive modes (2×2 matrix of zoom level × exploration/connection) and deliberately match them to task demands.

## The Four States

| State | Mode | Best For |
|-------|------|----------|
| **Dispersed** | Zoomed in + Exploring | Generating options, brainstorming |
| **Diversified** | Zoomed out + Exploring | Synthesis, seeing patterns, strategy |
| **Focused** | Zoomed out + Connecting | Refinement, quality control |
| **Biased** | Zoomed in + Connecting | Execution when direction is clear |

## Key Insight

All states exhaust when overstayed. Recovery comes from *movement*, not rest.

## Applications

| Context | State Flow |
|---------|------------|
| **Meetings** | Dispersed (ideation) → Diversified (prioritization) → Focused (decision) → Biased (action items) |
| **Learning** | Dispersed (exploration) → Diversified (connection-making) → Focused (drilling) → Biased (application) |
| **Debugging** | Biased (when you have a hypothesis), Dispersed (when you don't), Diversified (when symptoms don't cluster) |

## Self-Diagnosis

"I'm stuck" often means "I'm in the wrong state for this task."

**Ask:**
- What state am I in right now?
- What state does this task need?
- How do I shift?

## State Transitions

- **Dispersed → Diversified:** Step back, look for patterns in what you generated
- **Diversified → Focused:** Pick a direction, start refining
- **Focused → Biased:** Trust your refinement, execute
- **Biased → Dispersed:** Take a break, return with fresh eyes

## Red Flags

- Editing while drafting (wrong state for task)
- Brainstorming when you need to ship (avoiding Biased)
- Analysis paralysis (stuck in Diversified)
- Executing without direction (premature Biased)
